Vídeo Sem Flash

Permite mostrar um vídeo sem utilizar um plugin (flash/quicktime/etc)

É preciso ter cuidado ao seleccionar um formato de vídeo, o suporte para o formato de vídeo depende se o sistema operativo tem o codec instalado.

Depois de detectar se o browser tem vídeo, podemos simplesmente utilizar uma tag <video>

<video src="file.avi" width="320" height="240">

Também podemos utilizar vários formatos, o browser selecciona o formato que suporta:

<video width="320" height="240" controls>
  <source src="pr6.mp4"  type='video/mp4; codecs="avc1.42E01E, mp4a.40.2"'>
  <source src="pr6.webm" type='video/webm; codecs="vp8, vorbis"'>
  <source src="pr6.ogv"  type='video/ogg; codecs="theora, vorbis"'>
</video>

Vídeo Sem Flash

Podemos controlar o vídeo usando JavaScript

A tag <video> tem métodos play(), pause() e uma propriedade currentTime que podem ser utilizados para controlar o playback do vídeo. 

<script type="text/javascript">
  var v = document.getElementById("movie");
  v.onclick = function() {
    if (v.paused) {
      v.play();
    } else {
      v.pause();
    }
  };
</script>

Novos Formulários

Melhor UX

O html 5 define vários tipos de inputs que os dispositivos podem utilizar de forma a proporcionar uma melhor experiência para o utilizador.

Por exemplo, se o input for para um e-mail, um telemóvel pode mudar o teclado apresentado para mostrar @ ou mostrar os contactos disponíveis no telefone. Se for para um input para um telefone, o dispositivo pode mostrar um teclado só com números.

Validações automáticas

Sabendo o tipo dos campos, o browser pode implementar validações automáticas client-side, sem as implementarmos explicitamente.

Podemos usar já!

Todos os browsers modernos suportam estas extensões.

Caso o browser não suporte os novos tipos de campos, simplesmente mostra um input tradicional.

Canvas

Permite desenhar qualquer tipo de gráficos numa aplicação web

Deixamos de precisar de flash para renderizar gráficos

Permite utilizar WebGL para renderizar gráficos 3D directamente a partir de JavaScript. Os gráficos renderizados podem utilizar o GPU da placa gráfica instalada no computador do utilizador

Um <canvas> é um rectângulo na página onde podemos utilizar JavaScript para desenharmos o que quisermos

Exemplo Canvas

<script type="text/javascript">
  var canvasContext = document.getElementById("canvas").getContext("2d");
  canvasContext.fillRect(250, 25, 150, 100);
  canvasContext.beginPath();
  canvasContext.arc(450, 110, 100, Math.PI * 1/2, Math.PI * 3/2);
  canvasContext.lineCap = 'round';
  canvasContext.strokeStyle = 'rgba(255, 127, 0, 0.5)';
  canvasContext.stroke();
</script>

WebSockets

Permitem manter uma conexão TCP normal com o servidor

Não são enviados headers HTTP, é um protocolo simples

Utilizamos onopen, onmessage, send, etc para ter acesso à conexão.

Permite aplicações que mantêm uma conexão permanente com o servidor.

Não precisamos de utilizar long-polling para desenvolvermos aplicações real-time

Exemplos

Podemos ter aplicações web que avisam o utilizador quando o documento que ele está a editar foi alterado!

O cliente spreadd poderia manter uma ligação ao servidor e actualizar a interface assim que uma nova mensagem fosse recebida nessa conexão, sem ser necessário fazer polling ao servidor periodicamente.

WebSockets - Exemplo

        var socket = new WebSocket('ws://echo.websocket.org?encoding=text');
        socket.onopen = function(event) {
        	socket.send('Hello, Opensoft!');
        };

        socket.onmessage = function(event) { 
            logBox.value += 'Received echo: ' + event.data + '\n';
        }
        
        socket.onclose = function(event) { 
            logBox.value += 'Closed Connection\n';
        }        
    }

Guardar dados offline

Esta é uma das grandes vantagens de aplicações nativas

É comum precisarmos de guardar ficheiros de configuração, dados privados, etc no lado do cliente.

A única forma de manter dados de uma sessão para outra era guardar os dados no servidor e utilizar cookies

O html 5 define algumas APIs de forma a que as aplicações web possam ter esta c

Com estas APIs podemos:

  • Guardar dados do lado do cliente...
  • ...que são persistentes para além de um refresh da aplicação...
  • e nunca são transmitidos para o servidor

A única API suportada por todos os browsers modernos é a localStorage API

Exemplo LocalStorage

function saveGameState() {
      localStorage["game.in.progress"] = gGameInProgress;
      localStorage["game.selectedpiece"] = gSelectedPieceIndex;
      localStorage["game.selectedpiecehasmoved"] = true;
      localStorage["game.movecount"] = gMoveCount;
      return true;
}

Guardar dados Offline - Caching

Suporte para caching explicito

As aplicações passam a poder definir um ficheiro manifest onde são definidos os ficheiros que o browser deve guardar para que a aplicação funcione correctamente offline

<html manifest="/cache.manifest">
CACHE MANIFEST
/clock.css
/clock.js
/clock-face.jpg

Quando o browser detecta modificações neste ficheiro, os conteúdos guardados em cache são actualizados

Aplicações Offline

cache + localStorage = aplicações offline

Utilizando estas tecnologias, as aplicações web podem trabalhar offline como se fossem aplicações nativas!

O utilizador pode utilizar a aplicação offline e sincronizar o estado da aplicação quando estiver online novamente, caso seja um requisito da aplicação.

Manipulação da História

Alterar a história do browser

Podemos alterar o URL da janela sem que isso provoque um refresh total da página

Podemos ainda adicionar entradas à history do browser

Esta API permite-nos ter uma aplicação web que se porta como uma aplicação nativa. À medida que o utilizar navega na aplicação, toda a interface pode mudar sem que todos os elementos da página sejam renderizados novamente.

O url vai mudando, se o utilizador fizer refresh ou desligar o browser, mais tarde pode voltar ao URL e a aplicação pode estar tal como o utilizar a deixou.
